Required workflow
Vendor submits order manually or by bulk entry.
Manager checks whether the order is duplicate:
same customer
same phone
same address
same product
same day / recent previous day
If duplicate is found, system should clearly flag it.
If not duplicate, manager/admin assigns order to the nearest or selected rider based on service station / area.
Rider already carries vendor stock and attempts delivery.
Rider updates status and can add comments if there is an issue.
Vendor can see order status in real time and reply to comments if needed.
At the end of the day:
COD collection is calculated
delivery charge is deducted
payable amount to vendor is shown
admin/manager can release vendor payment
Vendor stock report must update automatically:
opening stock
delivered stock
remaining stock
Vendor can request COD release to their bank account.
Full reports, filters, analytics, and ticketing/help support must be included.
